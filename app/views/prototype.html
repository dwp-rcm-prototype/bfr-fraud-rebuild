Here are a list of the prototypes and the journey's explanations

  • Option 1
    Exit text to a button
  • Option 2

  • Option 3

  • (Current prototype) Testing exit

    Journey description:

    User does not select all 3 minimum options to get through details, gets re-directed to details-more, selects they dont know the missing selections and has an option to exit (example is to gov.uk)

  • Option 2 (Unburden box)

    Journey description:

    User does not select all 3 minimum options to get through details, gets re-directed to details-more, selects they dont know the missing selections and has an option to exit or continue.
    Continue takes them through to a page which shows the unburden box, then a review box.

  • Option 3 (Continue / exit test)

    Journey description:

    User does not select all 3 minimum options to get through details, gets re-directed to details-more, selects they dont know the missing selections and has an option to exit or continue.
    Continue takes them through to the details page - showing all options and a y/n for the "minor" data gathering fields.

  • Option 4 (Continue through service with input boxes on exit-page)

    Journey description:

    User goes straight to details-more, fills in form, leaves name, address or DOB empty and gets put to details-exit page to tell them "to continue you need to fill in details missed" or "no i dont know the answer but wish to continue anyway". Clicking fill in details shows two inputs on page, no makes user go to fraud types.
